Project Title
Sensor‑to‑Language Alerts and Voice Queries for Smart Collar MWD Health Monitoring
Sensor‑to‑Language: An NLP Framework for Interpretable Health Monitoring of Military Working Dogs
________________




Abstract
Monitoring the physiological state of Military Working Dogs (MWDs) is critical for mission success and canine welfare. Existing smart collars collect vital signs (heart rate, temperature, GPS) using wearable hardware and trigger threshold‑based alarms. However, these systems output raw data that require human interpretation, which can be slow or error‑prone in the field. We propose an NLP‑based extension that automatically translates sensor streams into natural language alerts and allows voice‑query access. Specifically, our system employs an anomaly detection layer to flag unusual readings and feeds structured sensor events to a transformer‑based natural language generation (NLG) model. This model produces concise human‑readable status reports (e.g. “Rex’s heart rate is high; possible stress detected”). Additionally, a speech interface (using Whisper) lets handlers query dog status hands‑free. We will implement and evaluate the architecture combining on-device preprocessing with cloud/NLP servers (HuggingFace models via LangChain). This multimodal sensor‑language framework aims to enhance situational awareness and response time by bridging the gap between raw wearable data and actionable insights for handlers.
________________




Introduction
Military Working Dogs (MWDs) are vital assets in defense operations, from explosive detection to search-and-rescue. During missions, MWDs often face extreme stressors (heat, physical exertion, combat noise), making real-time health monitoring essential for their safety and performance. Modern smart collars can sense heart rate, body temperature, and GPS location via on-collar modules, transmitting data to handlers. These collars use simple threshold alerts (beep or light) when vitals exceed limits. However, raw numerical outputs and basic alerts can be difficult for handlers to interpret, especially under duress. Handlers must manually correlate numbers with context, which risks delayed or incorrect decisions.
Recent advances in Natural Language Processing (NLP) suggest a promising solution: automatically translating sensor data into human-readable language. In healthcare, Natural Language Generation (NLG) systems have proven effective at summarizing vital-sign streams into text (e.g., ICU monitoring summaries). Large language models (LLMs) are increasingly applied to wearable sensor data, turning complex streams into insights. For example, Han Kim et al. (2024) evaluate LLMs for health prediction via wearable sensor data, while Yang et al. (2024) propose PhysioLLM for personalized health insights. Likewise, Zhang et al. (2025) introduce SensorLM, a transformer model trained to align wearable sensor traces with natural language captions. These works illustrate that bridging sensor streams and text is feasible and beneficial.
We extend this idea to the MWD domain. Our objective is to develop an end-to-end pipeline where smart collar readings are transformed into contextual alerts for handlers. Instead of cryptic numbers, handlers receive statements like: “Rex’s heart rate is elevated at 150 bpm with rising temperature; potential heat stress” (cross-referencing location and activity). Such alerts reduce cognitive load and enable faster action. Additionally, we incorporate a voice interface: using on-device speech-to-text (e.g., Whisper), a handler can ask “How is Rex doing?” and the system responds with a summarized health status. This hands-free query capability supports field conditions where using screens is impractical.
In summary, our project integrates embedded sensing, anomaly detection, and transformer-based NLG to make Military Working Dogs’ monitoring more intelligible. By converting multisensor inputs into natural language, we fill a key gap in current systems (raw data only), aiming to improve situational awareness and response time in the field.

Related Work
Wearable Sensor Data Analysis: Wearable health monitoring has been extensively studied. Banaee et al. (2013) review data mining for physiological signals (heart rate, respiration) in health systems, highlighting tasks like anomaly detection. However, they focus on feature-based analysis and not NLP output. Boursalie et al. (2018) survey mobile health pipelines, discussing ML on device but not text generation. Han Kim et al. (2024) benchmark LLMs for health prediction from wearables, and Yang et al. (2024) demonstrate an interactive system (PhysioLLM) integrating physiological data with LLMs to produce personalized health insights. These works suggest the viability of LLMs in sensor data domains but lack concrete NLP interface design for embedded monitoring.
Natural Language Generation from Sensors: Early NLG systems demonstrated converting structured data to text in healthcare and weather. Portet et al. (2012) developed BabyTalk, generating textual summaries of neonatal ICU signals. Reiter et al. (2005) created a weather forecasting NLG system converting numeric data into forecasts. These pioneers prove sensor-to-text is possible. More recently, Zhang et al. (2025) introduce SensorLM, a transformer-based foundation model trained on 59M+ hours of multimodal wearable data to produce natural language captions of sensor streams. SensorLM achieves zero-shot and few-shot tasks in healthcare data interpretation. Additionally, Li et al. (2024) introduce SensorLLM, which aligns motion sensor channels with descriptive text tokens so frozen LLMs can perform human activity recognition. While these models are impressive, they often rely on massive datasets, whereas our scenario involves constrained collars and domain-specific alerts.
Animal Vocalization and Behavioral Analysis: Audio processing of animal sounds is an active area. Molnár et al. (2008) analyzed 6,000 dog barks with ML, showing context‑specific acoustic features enable situational classification. More recently, Gómez‑Armenta et al. (2024) use deep learning to classify dog barks by context and emotion. Karaaslan et al. (2024) built a fully automated pipeline to detect and classify canine vocalizations for bioacoustics studies. Huang & Situ (2025) present a valence-arousal framework for pet vocalization, achieving high emotion prediction accuracy using a multi-task modeling approach. These works demonstrate audio‑based insights but do not integrate with physiological sensor data or generate natural language explanations.
Voice Interfaces and Retrieval-Augmented Models: Recent NLP tools enable conversational interfaces. Whisper (OpenAI) and Wav2Vec2 (Facebook) enable speech-to-text on device. Retrieval-Augmented Generation (RAG) models [Lewis et al., 2020] fuse LLMs with external knowledge for question answering, but are mainly applied to clinical Q&A rather than real-time sensor status. Some work explores RAG chatbots for medical queries. However, little exists on applying RAG to veterinary advice. For this project, RAG-like retrieval could support a vet-advice chatbot, but our focus will be on real-time status alerts rather than a broad knowledge system.
Gap Summary: In sum, while NLG and LLMs have been applied to wearable health data and canine audio signals have been classified, no existing system unifies these for MWD monitoring. Prior systems either analyze numeric streams (no language) or process dog barks (no sensor fusion). Our approach will bridge wearable sensors and NLP, a novel integration for military canine care.
________________




Gaps Identified
* No interpretability: Existing smart collar outputs raw numbers or simple alarms. There is no interpretable text explanation of the dog’s condition.
* Lack of NLG in animal health domain: Prior sensor-to-text work is focused on human healthcare (e.g., NICU). There is a gap in applying it to animal vitals.
* Isolated modalities: Dog audio analysis (bark classification) and physiological monitoring are separate fields. No system currently integrates barks with vitals in a single NLP framework.
* Limited real-time NLP: RAG and chatbots exist for medical Q&A, but not for on-field MWD monitoring. There is no voice-enabled interface for querying real-time dog status.
* Edge constraints: Large sensor-language models (e.g., SensorLM) require massive data and compute, impractical for our embedded scenario. We need a lightweight, deployable solution.
This project will directly address these gaps by generating contextual natural language alerts from collar data and providing a voice-query feature, all within an on-device/edge-friendly architecture.
________________




Objectives
1. Design an anomaly detection pipeline for MWD sensor data (heart rate, temperature, GPS) to detect events requiring alerts (e.g., tachycardia, hyperthermia).
2. Develop an NLG module that maps structured events (sensor anomalies) into concise, human-readable status reports and warnings.
3. Implement a speech interface using on-device speech-to-text (e.g. Whisper) allowing handlers to ask questions (e.g. “Is Rex OK?”) and receive verbal status responses.
4. Integrate system components on wearable and mobile/edge devices: sensor collection → data preprocessing → anomaly flagging → NLP model → alert output.
5. Evaluate performance by measuring alert accuracy (correctness of generated reports) and usability (handler response time with/without NLP alerts).

Proposed Architecture (Text Description)
The system consists of embedded sensing, data processing, and NLP/communication components. A text description suitable for diagramming:
* Smart Collar (Wearable Module):


   * Sensors: Heart Rate, Body Temperature, GPS.
   * Functions: Raw data acquisition at fixed intervals, preliminary noise filtering, packetization.
   * Transmission: Send processed sensor packets to base station via WiFi/BLE/LoRa.
   * Base Station / Edge Server:


      * Preprocessing Layer: Receives sensor streams; applies smoothing, computes features (e.g. short-term average HR, variance).
      * Anomaly Detection Module: Compares features to thresholds or runs a small ML model (e.g., Random Forest) to flag abnormal vitals (e.g., HR > 160 bpm, temp > 39°C).
Event Representation: When an anomaly is detected (or periodically), generate a structured event object, e.g.:
json
{
  "dog_id": "Rex",
  "timestamp": "2026-03-10T21:20:00Z",
  "heart_rate": 155,
  "temperature": 39.8,
  "gps": {"lat":33.8, "lon":151.2},
  "flags": ["high_hr", "high_temp"]
}
      *       * NLP Processing Layer:


         * Input: Structured event JSON.
         * Process: Template-based prompting or fine-tuned transformer (e.g., T5 or BART) for Natural Language Generation (NLG).
         * Example NLG pipeline: A rule/template (if HR > 150 bpm) may fill an input prompt like “dog heart rate high, generate alert.” Then a model (or few-shot GPT) expands to text.
         * Output: Natural language alert string, e.g.:
“Warning: Rex’s heart rate is 155 bpm and temperature is 39.8°C. Both are above normal. Possible heat stress. Location: Zone B.”
            * Voice Interaction Module:


               * Input: Handler voice query (e.g., “What’s Rex’s status?”).
               * Speech-to-Text: Use an ASR model (e.g., OpenAI Whisper) running on a smartphone or edge device.
               * NLP Intent Recognition: Parse query (e.g., by spaCy or a small LSTM) to identify dog ID and query intent (“health status”).
               * Query Response: The system retrieves the latest status (most recent alert or vitals) and generates a natural language response. For example:
“Rex’s vitals are normal. Heart rate 110 bpm, temperature 38.4°C, stable.”
               * Text-to-Speech (optional): Convert text answer to voice reply using a TTS system.
                  * User Interface:


                     * Mobile app / web dashboard showing alerts and status text.
                     * Voice I/O on device (earpiece or speaker for spoken responses).
                     * Open-source Tools (integrated at each stage):


                        * Collar firmware: Embedded scripts with communication stacks.
                        * LangChain / FastAPI: Orchestrate calls to LLMs and ASR, manage prompts.
                        * HuggingFace Transformers: Use models like T5-small for NLG and Wav2Vec2/Whisper for ASR.
                        * spaCy: For parsing queries or extracting key phrases if needed.
                        * TensorFlow Lite: (Optional) Deploy lightweight anomaly model on the wearable device if local inference is needed.
This modular architecture allows on-device sensing and off-device language processing, ensuring flexibility and performance.
________________




Open-Source Tools Identified
                        * HuggingFace Transformers – We will use pre-trained transformer models (T5, BART) for natural language generation of alerts. They provide easy fine-tuning and generation pipelines.
                        * OpenAI Whisper – An open-source speech-to-text model to enable voice queries. We can use a small Whisper model on a mobile device for offline ASR.
                        * LangChain – To orchestrate the NLP pipeline: manage prompts, chain sensor inputs to LLM outputs, and implement retrieval (if needed for context).
                        * spaCy – For NLP preprocessing in the voice query module, e.g., entity or intent recognition to identify dog name and query type.
                        * FastAPI – A lightweight Python web framework to host the NLP services (NLG, ASR) as APIs callable from the collar network or app.
                        * PyTorch/TensorFlow – Backend for running transformer models and any anomaly detection ML.
                        * TensorFlow Lite – (Optional) If needed, to run small ML models (like simple threshold or neural net) directly on the wearable hardware.
                        * Firmware libraries – For interfacing sensors and communication on the wearable board.
Each tool is well-supported and suited for its role: e.g., Whisper for high-quality transcription, Transformers for text generation, LangChain for system glue.
